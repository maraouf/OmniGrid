# Changelog

All notable changes to OmniGrid land here. Format adheres to
[Keep a Changelog](https://keepachangelog.com/en/1.1.0/), and the project
follows [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

Cadence (see `docs/RELEASE_PROCESS.md` for the full operator runbook):

- **`PATCH`** — CI bumps automatically on every successful deploy (one per shipped TODO item). The accumulating count between releases is the operator's "is it time to cut a release" signal.
- **`MINOR`** — operator-controlled. When a batch of PATCH-shipped items feels release-worthy, the operator hand-edits `MINOR` on the server (which resets `PATCH` to `0`) and writes a new `[X.Y.0]` section here listing the items that landed since the last MINOR.
- **`MAJOR`** — breaking changes only (DB migrations that aren't forward-compatible, env-var renames, `/api` contract breakage). Migration notes ship alongside the release in `notes/MIGRATIONS.md`.

Categories per release follow Keep a Changelog:

- **Added** — new features.
- **Changed** — changes in existing functionality.
- **Deprecated** — features marked for removal in a future release.
- **Removed** — features that were dropped this release.
- **Fixed** — bug fixes.
- **Security** — fixes for vulnerabilities.
- **Internal** — refactors, doc work, build / CI changes that don't touch user-facing behaviour. (Non-standard but useful for a homelab tool where most work is internal.)

Each entry references its TODO ID (`#NNN`) so the full implementation
detail in `notes/note_todo.txt` is one click away.

## [Unreleased]

Items that have shipped to the live deploy as a PATCH bump but haven't
yet been rolled into a numbered `MINOR` release. When the operator cuts
the next release, this whole block becomes the `[X.Y.0]` entry below.

### Added

- Host-drawer debug panel now exposes per-host counters & state. New "Counters & state" section in the Show debug data (admin) panel surfaces failure-state retry counters, per-provider pause + last-OK rows, time-series row counts (SNMP per-host, SNMP per-iface, node-exporter, ping), last-snapshot timestamp + size, and every probe-behaviour-affecting tunable (failure-pause thresholds, sample intervals, retention, SNMP timeout / concurrency, per-provider host-cache TTLs) live-resolved. Operators investigating "why is my host paused / why is my SNMP chart empty / what's the failure-pause threshold" no longer need to grep logs or open the SQLite DB.
- Snapshot-first render on the Hosts page. `/api/hosts/list` now pre-populates each row with last-known `host_*` fields from the persisted `host_snapshots` table, stamping `_stale_fields` / `_stale_ts` markers so the SPA's existing stale-rendering pipeline (dimmed values + "X minutes ago" tooltip) kicks in automatically. The per-host refresh still fans out in the background and upgrades each row from stale to fresh as it lands. Repeat visits paint instantly instead of waiting through the cold-cache cliff (was 10-20s on a fleet); first-ever visit (empty snapshot table) falls through to the legacy skeleton.
- APC UPS over-time charts in the host drawer (Output Load %, Battery %, Battery Temperature). Backend persists `load_percent` / `battery_percent` / `battery_temp_c` per SNMP probe — `host_snmp_samples` schema extended with three new columns and the sampler's INSERT trigger now fires on UPS hosts that have neither hrStorage NOR IF-MIB (basic Smart-UPS models). Three new chart cards render in the SNMP chart grid when the host has accumulated ≥ 2 ticks of the relevant metric; each card uses the same unified time-domain (1h / 6h / 24h / 7d picker) as every other drawer chart so spike alignment works across cards.
- `X-OmniGrid-Client-Id` request-correlation header for SSE self-filter. SPA generates a UUID per tab (persisted in `sessionStorage`), attached as a header on every API fetch by the global wrapper. Backend's `events.publish(event_type, payload, client_id=None)` extension stamps `client_id` into the event payload. SSE handlers in the SPA skip events whose `client_id` matches the local tab's id — i.e. "don't loop my own write back to me". Foundation for any "broadcast to other tabs but not to me" pattern; future write endpoints that publish events get the contract for free. First concrete consumer: `POST /api/settings` now publishes `settings:updated` with the originating client_id, so other tabs reload settings + /api/me on every save while the originator skips the redundant fetch.
- "Updated Xm ago" subtitle on every provider chip in the host drawer's Enabled-agents card. New `host_provider_last_ok` table tracks the most recent successful probe per (provider, host); UPSERTed by every `record_provider_outcome(... ok=True)` call so it lands automatically for every wired provider (Beszel, Pulse, NE, Webmin, Ping, SNMP). Surfaced via `provider_pause_state.<name>.last_ok_ts` on the host API row. Helps operators tell "fresh data" from "this chip is fine but we haven't heard from it in hours".
- Compact stale display when an entire provider's worth of metrics has been served from snapshot. New `isAllStale(h)` helper (≥6 stale fields) widens the drawer banner to a more prominent variant AND suppresses the per-field warning triangles via a `.host-drawer.bulk-stale` CSS scope, so the operator gets one clear top-of-drawer signal instead of every `<dl>` row carrying its own triangle. The `.stale` class still fires (dimmed + dashed-underline + tooltip on hover) so individual cached values remain visually distinct.
- Per-(provider, host) auto-pause + manual resume now generic across EVERY provider — Beszel, Pulse, node-exporter, Webmin, Ping, SNMP. Cool-down still throttles each individual failure (where present), but after N consecutive failed rounds (configurable per provider via `<PROVIDER>_FAILURE_PAUSE_ROUNDS` / `tuning_<provider>_failure_pause_rounds`, default 5; Ping default 0 because alive=False is data not a fault) the (provider, host) pair gets MARKED auto-paused; the chip in the host drawer flips to amber Paused state with a tooltip showing the consecutive-failure count + last error, and a Resume button (admin-only) clears the marker so the next probe runs immediately. Distinct from the host-level `host_failure_state` (per-host) — this is per-provider, so a host with broken SNMP but healthy Beszel shows ONE chip paused, not the whole row. Hub-based providers (Beszel/Pulse) gate failure recording on hub-fetch-OK so a global hub blip can't cascade-pause every host. New generic `record_provider_outcome(host_id, provider, ok, error="", round_threshold=0)` helper turns probe-site instrumentation into a one-liner; adding a new provider in future is the documented six-step contract: TUNABLES + SettingsIn + relocatedTuningKeys + admin panel + i18n + record_provider_outcome at the probe site.
- 32-bit counter degraded badge on the host drawer's network-iface chip strip. SNMP devices that don't expose ifHCInOctets fall back to 32-bit ifInOctets (wraps every ~17s on a 1 Gbps link at line rate). Backend already tags rows with `counter_width: 32 | 64`; frontend now renders a small amber "32" chip next to the iface name with a tooltip explaining that the throughput chart can silently miscount under-wrap traffic.
- Settings GET version int (`/api/settings/version`) for cheap cross-tab change detection. `set_setting` bumps a monotonic `_settings_version` row on every write; SPA can poll the bare version endpoint instead of re-fetching the full settings blob.

### Changed

- Time-range picker (1h / 6h / 24h / 7d) on the host drawer now disables its buttons while the underlying loaders are in flight. Pre-fix operators could rapid-click between ranges while the first fetch was still resolving — on slow networks responses landed out of order and presented inconsistent chart data. Picker now awaits every loader via `Promise.allSettled` and reuses the picker's busy flag for `:disabled` binding; one failing loader doesn't lock the picker.
- Host / item / node drawers now lock body-scroll while open. Previously, scroll-wheel input over the page background could drift the underlying view while the operator interacted with the drawer. Affects all three drawer surfaces: Hosts, Stacks/Services items, and the Nodes-view drawer.
- "Counters & state" section in the host drawer's debug panel moved to the first position. Operators investigating a host issue almost always want to see failure-state + per-provider pause status + sample row counts before wading through the raw provider JSON.
- APC UPS card in the host drawer now renders Battery state as a coloured pill (Normal → green, Low → amber, In fault → red, Unknown → grey) instead of the raw `battery-normal` / `battery-low` enum string. Stale styling (dashed underline + dimmed value + "X minutes ago" tooltip) wired across every UPS card field — header status pill, battery %, output load %, runtime remaining, battery temperature, battery state — so when the SNMP provider goes offline operators see the cached values flagged as stale instead of an empty card.
- Host-drawer charts now share a unified time x-axis. New `_drawerTimeDomain()` helper anchors every chart to `[now - rangeMs, now]` based on the picker's selected window (1h / 6h / 24h / 7d). Every drawer chart — Beszel / NE CPU%, Mem%, Disk%, Net I/O, Disk I/O, Load, GPU, Swap, Temperature, the Ping RTT card, and every SNMP card (CPU per-core, load, memory stacked-area, total throughput, per-port throughput, per-port utilization, printer pages) — now uses time-based x-stepping instead of computing x by sample index. The same horizontal pixel position now means the same wall-clock time across every card on the drawer, so operators can visually align spikes across providers. Sparse providers (e.g. node-exporter returning 4 samples in an hour) start mid-axis where their earliest sample landed instead of stretching to fill the full width — making "I have 60 minutes of data" visually distinct from "I only have 38 minutes". X-axis labels (`xAxisFromSeries` / `snmpXAxis`) emit ticks at evenly-spaced positions across the picker window instead of through the actual sample series, so the same label rhythm holds across cards.
- Services view's Node column now renders topology-style pills (host name + state-coloured dot, green for running) instead of the previous comma-separated text. Reuses `topologyGroups(item)` already used by the Stacks view's per-item rows so a single helper drives both surfaces, and pills wrap onto multiple lines via `.topology-map`'s `flex-wrap: wrap` so a service with replicas on many nodes shows them all without horizontal scroll.
- SNMP throughput / per-port throughput / per-port utilization charts now render genuine null gaps as visual breaks instead of straight-line bridges. New `_snmpPathGapped` helper emits an SVG `path d` with `M` (moveto) at every null so a single `<path>` renders as multiple disconnected segments. Counter-rate wraps / reboots / timeouts now look unmistakably different from "host idle" zero segments.
- Module-load INFO line in `logic/snmp.py` reports which pysnmp walk function the resolver picked (`bulkWalkCmd` vs legacy `bulkCmd`) plus the active pysnmp namespace, so operators debugging "walk hangs" / "walk truncates" can confirm the active code path from the first SNMP-tagged log line at startup.
- `host:history_appended` SSE handler debounced 200 ms inside the per-host filter so a burst of events for the same open drawer host within one sampler tick collapses to a single `loadHostHistory` call instead of N redundant fetches.

### Fixed

- SNMP / Webmin cool-down log lines no longer flag as ERROR in Admin → Logs and now include the resolved probe target. Pre-fix the log read `[hosts] snmp probe failed for 'idrac': in cool-down (192s remaining)` — the persistent-log severity classifier matched the word "failed" as ERROR even though cool-down is a skip (not a real failure), so the ERROR bucket flooded with red on every drawer poll while a host was unreachable. Now branched: cool-down skips read `skipped (cool-down) for 'idrac' → idrac.example.com: …` (INFO severity); real failures keep "failed" (ERROR). Both include the resolved target alongside the host id so operators tracing back-off can see what hostname / IP was actually being probed.
- Printer / UPS card freshness banner timestamp now matches the SNMP chart freshness. Pre-fix the per-host snapshot save fired on every drawer poll (~every 15s), refreshing the snapshot's `ts` to "now" even when every field came from snapshot fallback (no fresh live probe). Result: the cached-data banner showed "Last live data 3m ago" while the SNMP chart correctly reported "Last sample 6m ago". Snapshot save now skips when the merged dict is entirely from fallback — at least one field must be live (not in `_stale_fields`) for the persistence ts to advance.
- "No data from any enabled provider — could not match this host" banner no longer fires on hosts where the provider was auto-paused. Pre-fix two banners showed simultaneously with conflicting copy ("could not match" vs "auto-paused"); the auto-paused banner now stands alone since its message accurately explains why no live data is reaching the host.
- "Collecting first samples" hint on SNMP charts no longer fires when the host already has stale historical samples (e.g. last sample 2h ago). Pre-fix the hint promised data "after the next sampler tick" even when no tick was actually imminent. Hint now only shows when zero history exists; the freshness banner above ("Last sample N ago") carries the messaging when stale data exists. Hint copy also reflects the SNMP-specific sampling cadence (`tuning_snmp_sample_interval_seconds`) when configured, instead of always using the global Beszel/NE cadence.
- Pulse and Beszel probes no longer accumulate "host not found in hub map" failures on hosts that don't have those providers configured. Pre-fix the lookup keys fell through to the host_id when no `pulse_name` / `beszel_name` alias was set — every host got probed against the hubs and the always-missing lookup auto-paused providers on hosts that were never configured for them. Strict gate now requires explicit alias to opt the host in. Existing orphan paused rows clear automatically on next host-config save (`_sweep_orphan_provider_state_rows`) or via the per-chip Resume button.
- Pulse + Beszel probe failures now log to Admin → Logs. Pre-fix the exception was returned silently in the API response's `error` field — operators saw "Pulse: down" on the Hosts page with no log entry explaining why. Both probes now print a `[pulse] probe failed: ...` / `[beszel] probe failed: ...` line with the exception class + message + connection target before returning.
- Curated SNMP-only hosts (UPSes / managed switches / printers) now get persisted snapshots from the per-host probe path, not just the legacy gather path. Pre-fix, snapshots were only written by the Stacks/Services view's `_gather_impl` (which builds nodes_info from Swarm-node hostnames) — non-Swarm hosts never appeared, so the snapshot fallback had nothing to restore when SNMP stopped returning data → operator-reported "UPS card disappears even though SNMP says Updated 7m ago". Now `_merge_one_host` writes the snapshot when at least one host_* field is meaningful.
- Charts cropped from the right on first drawer open. The history-cache gate didn't refresh data fetched at an earlier "now", so the unified time-domain (anchored on `Date.now()` at render time) clamped every cached point to the left edge of every chart until the operator clicked a different range filter. Now re-fetches when the cache is older than 30s. Applies to Beszel/NE history, ping history, per-host SNMP history, and per-iface SNMP history.
- Per-port utilization chart now renders on SNMP hosts whose agent doesn't expose `ifHighSpeed` (printers / embedded gear). Pre-fix the chart's gate required a known link speed for the divisor — printers typically don't report it so the card stayed at "Collecting data…" forever. New 100 Mbps fallback divisor with a per-iface asterisk + tooltip ("Link speed not reported by SNMP agent — utilization computed against an assumed 100 Mbps") so operators read the percentages as approximate, not authoritative. Managed switches still get authoritative percentages without the asterisk.
- UPS info card stayed hidden on UPS hosts whose SNMP probe extracted battery / temperature / load but not the upsBasicOutputStatus OID (basic Smart-UPS models, or transient OID failures). The card gate now fires on ANY UPS field present (live or stale) instead of just `host_ups_status`. Eight UPS / hardware fields added to the in-place-reconcile whitelist so the SPA explicitly tracks them across polls instead of falling through to the generic-assign path that couldn't clear stale values or surface snapshot fallbacks.
- Printer info card stayed visible with cached values when the SNMP provider goes offline. Section gate accepts stale-fallback fields, each row inside binds the standard `.stale` styling + "X minutes ago" tooltip, three printer fields added to the in-place-reconcile whitelist. Mirrors the UPS-card pattern shipped earlier.
- Stale banner ("Showing cached data — provider unavailable") now lists the actual field names the snapshot fallback restored. Pre-fix the banner read "1 field(s) restored" with no way to tell which field — fields rendered off-screen or deprecated bind points produced phantom banners that looked wrong. Listed inline below the banner body and in the banner's title attribute for transparency.
- Beszel history charts now show the full requested window. Pre-fix `fetch_system_history` always queried Beszel's `1m` aggregation tier (retained for ~1 hour on the hub) regardless of the picker's selected window — so a 24h pick only returned the last hour of samples even though longer-window aggregations were sitting on the hub. New `_pick_stat_type(hours)` selects the right tier per window: `1m` for ≤1h, `10m` for ≤12h, `20m` for ≤48h, `120m` for the 7d range. Operators with default Beszel hub schema get the full window automatically; operators on a custom-trimmed schema can fall back via the explicit `stat_type` kwarg.
- Host-drawer pause-banner + Resume-button consistency pass. (a) The per-provider "N providers auto-paused" rollup banner now hides when the whole-host sampling-paused banner is showing — pre-fix both rendered simultaneously with the whole-host Resume button disabled while Resume all (N) was enabled, contradictory state operators reported as confusing. (b) Whole-host Resume sampling now ALSO clears every paused provider for the host in one action — operators no longer have to click both buttons to fully recover a host whose every layer paused. (c) Every Resume button (per-chip, top rollup, bottom rollup, whole-host) now shares one `canResume*` predicate family — the per-chip-disabled-while-top-rollup-enabled pattern operators reported is impossible by construction. (d) Whole-host Resume sampling gained a 30-second safety timer mirroring the per-provider Resume's existing safety; `openHostDrawer` now force-clears `h._resumeBusy` on entry. Stuck busy flags from a prior click that never resolved (network freeze, browser killed the await) now recover within 30s OR on the next drawer reopen. (e) The Enabled-agents card now hides chips for any provider that's globally disabled at the fleet level — a stale SNMP chip with a Paused state on a fleet where SNMP is globally off was the chip-render hole the previous pass didn't fully close.
- Time-range picker (1h / 6h / 24h / 7d) now renders on the host drawer for SNMP-only hosts (managed switches / UPSes / NVRs / embedded routers). Pre-fix the picker's gate required Beszel / NE / Ping so SNMP-only hosts saw their charts stuck at the initial 1h window with no way to switch range. Pairs with the earlier change that wired `setHostHistoryRange` to also refresh SNMP series.
- Per-(provider, host) auto-pause SSE event now publishes the BARE `host_id` plus a separate `provider` field — previously leaked the prefixed `<provider>:<host_id>` key as `host_id`, breaking the SPA's `refreshHostRow(id)` lookup with a silent 404. Drawer chips now flip to amber within one SSE round-trip on auto-pause instead of waiting up to 15s for the next poll. New `_split_failure_key()` helper in `logic/host_metrics_sampler.py` + `_PROVIDER_PREFIXES` frozenset.
- SNMP and Webmin successful probes now route through `record_provider_outcome(... ok=True)` so `host_provider_last_ok` UPSERTs land — the chip's "Updated Xm ago" subtitle was invisible-forever for those two providers because they bypassed the helper and called bare `_clear_failure` directly.
- Resume endpoint (`POST /api/hosts/{id}/provider/{name}/resume`) now threads `request: Request` and passes `client_id=_request_client_id(request)` into the `host:failure_state_changed` publish — the originating tab no longer flickers from amber → grey-loading → ok via a redundant SSE-driven refresh after its optimistic-clear has already updated the UI.
- Multi-field admin Saves through `POST /api/settings` now collapse N `_settings_version` bumps into ONE via the new `defer_settings_version_bump()` context manager. Previous behaviour: a single Save with 30 fields fired 30 `settings:updated` SSE events, prompting other tabs to reload `/api/settings` + `/api/me` thirty times. Post-fix: one bump, one event, one reload.
- Auto-pause Apprise notify body branches on whole-host vs per-(provider, host) pause. Per-provider pauses now read "Resume manually from the SNMP chip in the host drawer" instead of the misleading "host drawer's banner" (the banner only handles whole-host resume); title "Provider paused: web01 (snmp)" instead of "Host sampling paused: snmp:web01".
- Orphan `host_failure_state` + `host_provider_last_ok` rows are now swept by `api_hosts_config_set` after every Save — when the operator deletes a host from `hosts_config`, the matching prefixed/bare rows in both tables are deleted in the same request. Without the sweep, orphans accumulated forever and degraded the LIKE-scan performance of `_provider_pause_state_for_host`. New helper `_sweep_orphan_provider_state_rows(live_ids)`.
- `/api/hosts/list` performance fix: replaced per-host leading-wildcard `LIKE '%:host_id'` SELECTs (which prevented index use → ~480k row comparisons per request on a 200-host fleet × 1200-row table) with a single full-table scan cached for 5s. New helpers `_build_provider_state_index()`, `_get_provider_state_index()`, `_invalidate_provider_state_cache()`. Resume / sweep paths invalidate explicitly so operator actions reflect immediately.
- `_isSelfEvent` parsed `e.data` and then every handler parsed it again. New `_unwrapEventOrNull(e)` helper returns the parsed event object (or null when self-filter wins), eliminating the per-handler double-parse for new handlers. Existing handlers stay on `_isSelfEvent` for now (migration is a future cleanup).
- `op:created` / `op:updated` / `op:completed` console.log statements moved AFTER the `_isSelfEvent` filter check — pre-fix every op the originating tab triggered logged to the browser console even though `_handleOpEvent` then filtered it out (log spam on self-publish).
- Cool-down detection in the auto-pause counter path now consults a structured `result["skipped_cooldown"]` boolean from the probe-result dict (set by `logic/webmin.py:probe_webmin` and `logic/snmp.py:probe_snmp`) instead of fragile substring-matching the message text. Substring matching breaks silently if a future i18n pass or typo changes the wording.
- Single `attachAllHeaders(init)` helper in `static/js/auth-fetch.js` replaces the two-call `attachClientId` + `attachCsrf` chain. One Headers construction + every header set + assigned back ONCE; eliminates the order-coupling bug class where a future refactor making one helper mutate-in-place would silently drop the other's header on the CSRF retry path.
- Resume-button styling distinct from the paused chip — pre-fix the button used `class="chip"` which visually nested inside the paused chip, making the interactive boundary ambiguous. Now renders as an explicit button with border, hover state, and a small play-icon.
- `docs/guidelines/api.md` SSE event-types table reconciled — `settings:updated` added, the stale `host:row_updated` row removed (the event was retired earlier in 1.2.x to break the SSE infinite-loop pattern, but the doc table still listed it); `host:failure_state_changed` payload doc updated to reflect the bare `host_id` + optional `provider` field.
- Defensive `.get(key, default)` swap across every `_TOTP_POLICY_DEFAULTS[...]` and `_NOTIFY_EVENT_DEFAULTS[...]` direct subscript in `main.py` (~12 sites). A future registry-shrink would have raised KeyError on every `/api/settings` GET → 500; defensive defaults preserve current behaviour.
- Dedicated SNMP sample interval (`SNMP_SAMPLE_INTERVAL_SECONDS` / `tuning_snmp_sample_interval_seconds`). Default 0 inherits the global stats sample interval; any value 30..3600 overrides for SNMP probes only. Sampler tracks the last SNMP tick and fires only when the interval elapses, so SNMP gear can run on a different cadence than Beszel / node-exporter agents (printers can poll hourly, switches every minute). Surfaced in both Admin → Config and Settings → Providers → SNMP (panel block extended from 2 to 6 SNMP tunables).
- "Edit" button (admin-only) on the host drawer header. Single click closes the drawer, switches to the Admin → Hosts view, and scrolls the matching curated row into view with its config expanded — same chain the System card's "+ Add URL" link uses.
- Total Throughput chart (SNMP) gains a static-rate headline above the chart line — "↓ rx ↑ tx" in 16 px semibold mono so operators see the current rate without parsing the chart trend. Same prominence as the printer lifetime page count.
- Hardware Inventory card in the host drawer surfaces SNMP-reported model / serial / firmware separately from the asset-inventory rows, so operators can spot hardware swaps not yet recorded in inventory.
- GPU chart cards in the host drawer for hosts with discrete GPUs (Beszel agent reports `stats.g`). Three new cards: GPU Power Draw (W, average across GPUs), GPU Usage (%, average utilization), GPU VRAM (% of VRAM used, legend shows used / total bytes). Cards hidden on non-GPU hosts. Backend normalises Beszel's inconsistent unit convention (used in GB, total in MB) to bytes uniformly. History endpoint emits per-tick aggregates so polylines populate immediately.
- Switch-specific SNMP charts on the host drawer. Total throughput chart with in / out polylines from cumulative ifHCInOctets / ifHCOutOctets sums, with skip-don't-synthesize on counter wrap or reboot. Per-port throughput multi-line chart showing the top 5 interfaces by current rate (solid = in, dashed = out, color-cycled). Per-port utilization heatmap showing every interface as a colored chip with name + util %, color-graded against link capacity (green < 50 % < amber < 85 % < red, grey for unknown speed / idle). Uptime trend with reboot detection — drawer's System card uptime row renders a "Rebooted Xh ago" badge when the SNMP uptime counter walks backwards. Per-iface counter persistence in a new `host_snmp_iface_samples` table; ifHighSpeed walk added to the SNMP probe.
- Printer pages-printed sparkline chart for hosts that report Printer-MIB `prtMarkerLifeCount` (HP, Brother, Canon, Epson, Xerox, etc.). Pages-per-day rate derived from adjacent samples with absurd-rate guards. Lifetime page count repositioned inside the Printer card body at 18 px semibold mono with a "pages printed (lifetime)" caption.
- Hardware inventory rows in the host drawer's Hardware card now surface SNMP-reported model / serial / firmware (entPhysicalTable + vendor MIB walks), distinct from the asset-inventory rows so operators can spot hardware swaps not yet recorded in inventory.
- Faded amber warning triangle (⚠) prefix on every stale text element. Existing dashed-underline + dim opacity weren't immediately recognisable as "cached data". Triangle uses Unicode `⚠` at 0.85 em, scoped via `.stale:not(.stat-bar)::before` so it doesn't double-render inside stat-bars.

### Changed

- Per-port utilization chart converted from chip-strip heatmap to a true LINE CHART. Top 5 interfaces by current rate, plotted as utilization % of link capacity over time. Y-axis 0-100%, x-axis timestamps, color-cycled lines. Spinner during warm-up requires both known link speed AND ≥ 2 ticks of bps history (otherwise an empty chart with 0% values rendered instead of the spinner).
- Beszel Load avg chart now renders as % of cores (capped at 100) instead of raw load values like 0.18 / 0.22, matching the SNMP Load chart treatment. Cores resolved per-tick from the `cpus` array length in each system_stats row. Chart Y-axis uses standard 0 / 50 / 100, title chip switched from `load` to `%`.
- Chart unit chips are now DYNAMIC for byte-rate / byte-magnitude charts. New `unitForBytes(n)` helper picks the unit symbol (B / KB / MB / GB / TB) that `fmtBytes` / `fmtBytesAt` would render for the chart's max value, so the chip + legend + Y-axis all share one unit family. Pre-fix a static `B/s` chip read as a contradiction next to a `12 MB/s` legend value.
- Chart legend + Y-axis values across Network / Disk-IO / Bandwidth / Total throughput / Per-port throughput / SNMP Memory now use `fmtBytesAt(value, max)` so every formatted value uses ONE unit family. Pre-fix the legend rendered "0 B/s" next to "19 MB/s" (per-value autoscale).
- "Collecting data..." spinner pattern landed on EVERY chart card during warm-up — Beszel/NE side (CPU / Memory / Disk / Network / Disk I/O / Bandwidth / Load / Swap / Temperature / GPU x3) and SNMP side (CPU per-core / Load / Memory / Total throughput / Per-port throughput / Per-port utilization). Each chart body wrapped in an `x-if` gate with a sibling spinner block, so an empty grid never renders during the warm-up window.
- "Updated Xs ago" freshness label suppressed on permanently-flat charts. Pre-fix the label rendered whenever the polling loop touched `loadedAt` even though the series was empty / all-zero — operator read "Updated 2s ago" as fresh data on a chart with nothing to look at. Gate now requires ≥ 2 history points AND non-zero data across the canonical metric keys.
- Pages-printed sparkline chart REMOVED entirely. The Printer card already surfaces the lifetime page count as an 18 px headline; the pages-per-day rate sparkline added marginal value for the cost of a permanently-spinner card on idle printers.
- Printer card now shows freshness banner ("Last sample Xm ago" in orange) whenever SNMP history exists, not just when the snapshot fallback fires. When fallback DOES fire, the banner widens to "Showing cached data — last refreshed Xm ago".
- Provider icons + text labels in chips visually centered.
