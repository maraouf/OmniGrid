# Changelog

All notable changes to OmniGrid land here. Format adheres to
[Keep a Changelog](https://keepachangelog.com/en/1.1.0/), and the project
follows [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

Cadence (see `docs/RELEASE_PROCESS.md` for the full operator runbook):

- **`PATCH`** — CI bumps automatically on every successful deploy (one per shipped TODO item). The accumulating count between releases is the operator's "is it time to cut a release" signal.
- **`MINOR`** — operator-controlled. When a batch of PATCH-shipped items feels release-worthy, the operator hand-edits `MINOR` on the server (which resets `PATCH` to `0`) and writes a new `[X.Y.0]` section here listing the items that landed since the last MINOR.
- **`MAJOR`** — breaking changes only (DB migrations that aren't forward-compatible, env-var renames, `/api` contract breakage). Migration notes ship alongside the release in `notes/MIGRATIONS.md`.

Categories per release follow Keep a Changelog:

- **Added** — new features.
- **Changed** — changes in existing functionality.
- **Deprecated** — features marked for removal in a future release.
- **Removed** — features that were dropped this release.
- **Fixed** — bug fixes.
- **Security** — fixes for vulnerabilities.
- **Internal** — refactors, doc work, build / CI changes that don't touch user-facing behaviour. (Non-standard but useful for a homelab tool where most work is internal.)

Each entry references its TODO ID (`#NNN`) so the full implementation
detail in `notes/note_todo.txt` is one click away.

## [Unreleased]

Items that have shipped to the live deploy as a PATCH bump but haven't

### Fixed

- SNMP walks no longer crash on pysnmp 7.x (#687). Pre-fix every `_snmp_walk` call raised `TypeError: 'async for' requires an object with __aiter__ method, got coroutine` because pysnmp 7.x's `bulk_cmd` returns a coroutine (single response), not an async iterator. The Admin → Logs view spammed errors continuously for every walk on every SNMP-probed host (~17 walks × N hosts × every gather cadence). The import resolver now picks up `bulk_walk_cmd` / `walk_cmd` (7.x walk APIs) when present; `_snmp_walk` uses `bulkWalkCmd or bulkCmd` so pysnmp ≤6.x's legacy iterator-style `bulkCmd` still works as the fallback.

### Added

- Vendor-specific SNMP MIB support — Dell DELL-RAC-MIB (iDRAC), Cisco CISCO-MEMORY-POOL-MIB / CISCO-PROCESS-MIB / SG300 CISCO-PRODUCTS-MIB, APC PowerNet-MIB (Smart-UPS family), UCD-SNMP-MIB (Linux net-snmp), SYNOLOGY-MIB (DSM NAS), Ubiquiti UniFi sysDescr "MODEL, FIRMWARE" parser (#682–#686). Pre-fix iDRAC / Cisco SG300 / APC Smart UPS / Asus DD-WRT / Synology / UBNT all returned only sysName + sysDescr from the standard surfaces — devices with rich data hidden under vendor-private MIBs OR (for UBNT) embedded in sysDescr looked sparse in the host drawer. New `extract_vendor_info` helper that NEVER overwrites earlier-extractor values reads vendor-specific OIDs and populates `host_model` / `host_serial` / `host_firmware` / `host_health` (Dell global system status / Synology system status), `host_battery_percent` / `host_battery_runtime_s` / `host_battery_temp_c` / `host_battery_status` / `host_load_percent` / `host_ups_status` (APC), `host_load_1m` / `_5m` / `_15m` (UCD), `host_temp_c` / `host_upgrade_status` (Synology). Verbose `raw` debug payload extended with per-vendor blocks + matching `walk_summary` row counts. The "any data" gate now passes on a successful Dell / Cisco / APC / UCD / Synology vendor probe so devices that respond ONLY to vendor MIBs don't trip the cool-down. `extract_sys_info` extended with sysContact / sysLocation reads + Ubiquiti-style sysDescr parsing.
- SNMP-aware `host_metrics_sampler` — auto-pause for permanently-unreachable SNMP hosts, mirroring the NE-mapped-host pattern (#679). New `_probe_one_snmp(host, sem)` runs on the same tick as the NE block when `"snmp" in active_sources`; failure-state rows are keyed `snmp:<host_id>` so SNMP and NE failure streaks stay independent without a schema change. New `curated_snmp_hosts()` walker in `logic/db.py` returns opt-in hosts from `hosts_config[].snmp.enabled === True`. Concurrency cap from `tuning_snmp_concurrency`. Phase-1: SNMP-prefixed pause is logged but the drawer's "sampling paused" badge still keys on bare host_id — Phase-2 will surface SNMP-specific indication.
- Stat-bar warn / crit thresholds now operator-tunable via TUNABLES (#695). New `tuning_stat_bar_warn_pct` (default 60, range 30..90) and `tuning_stat_bar_crit_pct` (default 85, range 50..99). SPA's `barLevel` / `barColor` helpers read `me.client_config.stat_bar_warn_pct` / `_crit_pct` per-call so an Admin → Config save lands on the next render. Wired through the full 6-step audit (TUNABLES, SettingsIn, `client_config` channel, tuningKeys, i18n, env_example).
- Hosts-page header provider chips are now clickable filters (#680). Each provider chip (beszel / pulse / node-exporter / webmin / ping / snmp) renders the SAME mono SVG icon (CSS mask-image + currentColor) the Admin → Hosts collapsed-card chip strip uses — replacing the previous text-only labels. Clicking a chip toggles it into the active filter set with multi-select OR semantics: ticking `pulse` and `snmp` shows hosts mapped to EITHER provider. A new "All" pill on the left clears the filter; a new "No provider" pill on the right (only visible when at least one curated row has no provider mapped) filters to inventory-only hosts that have no live data source. Filter state persists per-tab via `sessionStorage`. New `.chip-active` CSS rule adds a subtle inset ring + slight brightness lift so selected chips visibly distinguish from inactive ones, regardless of the per-provider colour. Chips render as `<button>` elements with proper button reset CSS so they look identical to the previous `<span>` chips while being keyboard-focusable.
- `probe_snmp` now reads ENTITY-MIB physical-component walks + sysContact / sysLocation (#681). Vendor-agnostic device identification — works on Cisco / Dell iDRAC / HP / Juniper / virtually every enterprise switch / server / UPS / NVR. New `host_model` / `host_serial` / `host_firmware` / `host_location` / `host_contact` fields land in the normalised stats; the verbose `raw` debug payload (#675) gains `entity_count` + `entity_rows` (per-physical-entry summary aligned by entPhysicalIndex) so operators can see every chassis / slot / supply / fan / port the agent reported. The "any data" gate that arms the per-(host, port) cool-down now also clears on ENTITY-MIB success — some agents are locked down to ENTITY-MIB only and previously fell through to the no-response branch even when they answered. Particularly addresses the Cisco SG300 / Dell iDRAC case where the device exposes rich identification but doesn't fully implement Host Resources MIB (no hrStorage / hrProcessorLoad).
- New `tuning_snmp_unreachable_cooldown_seconds` knob — dedicated SNMP-specific cool-down dial (NIT-004 / #678). Default 300 s (parity with the legacy auth-failure cooldown so behaviour is unchanged on first deploy); range 30..3600. Operators debugging "SNMP timing out" can now tune the SNMP-specific dial directly instead of reaching for the auth-failure knob (which legitimately has different semantics — auth-failure is for credential-lockout backoff, SNMP cool-down is for short-circuit on UDP timeouts where there's no auth challenge to lock out against).

### Fixed

- SNMP debug-panel raw payload expanded to include parsed walk data (#675). Pre-fix the SNMP raw debug box surfaced only connection params (`target`, `community`, `version`, `port`, `v3_user`) + a `hosts_keys` list — operators inspecting why a Cisco SG300 wasn't producing host_* fields had no visibility into WHICH OIDs the agent actually answered. Operators can confirm at a glance which OID families their device answers without having to drop to `snmpwalk` on the manager.
- Dockerfile OCI label now cross-references `deploy.yml`'s `--label` override (LOW-003 / #672). The committed `image.source="https://git.example.com/<owner>/OmniGrid"` is a deliberate placeholder for the public-shippable Dockerfile; the deploy pipeline stamps the operator's real git host into the actual built image via `--build-arg SOURCE_URL`. Multi-line comment now documents this so future contributors don't think the label is stale and re-introduce operator-private hostnames into the public surface.
- `_clean_host_snmp` documents the intentional omission-equals-disabled contract (LOW-004 / #673). The opt-in pattern (#654) made the read-side gate `enabled is True`, which interprets a missing `enabled` key as OFF — so the SPA's strip-blanks pattern that drops `enabled: false` to keep JSON tight stays correct. Comment block makes the contract explicit so a future contributor doesn't re-introduce a default-true fallback.

### Changed

- SnmpEngine reused across `probe_snmp` calls via a module-level singleton (LOW-002 / #671). pysnmp HLAPI engines carry per-engine state (USM key cache, message-id state); pre-fix every probe constructed a fresh `SnmpEngine()`, wasting ~2-3 ms × N concurrent × every gather cadence. Lazy-initialised under an `asyncio.Lock` so the first concurrent burst doesn't race two allocations. Reused across every probe; not closed explicitly (pysnmp HLAPI doesn't expose a clean shutdown path; lifespan teardown drops the process anyway).
- WebAuthn `_ALLOWED_TRANSPORTS` (frozenset) and `_TRANSPORT_ORDER` (tuple) hoisted from per-credential loops in `verify_registration` / `make_authentication_options` to module scope (LOW-001 / #670). Both used to reallocate per call. Module-level constants now allocate once per process — micro-optimisation, but a defensible cleanup that simplifies future reads of the loop bodies.
- `.forgejo/workflows/deploy.yml`'s `docker login` step now redirects stderr to `/dev/null` (LOW-005 / #674). Defence-in-depth: token is bounded to `secrets.REGISTRY_TOKEN` (repo-admin-only) already, but stripping login's stderr means no exposure surface even if a future docker version's failure mode emits credentials there. The outer step's `tee /tmp/deploy.log` can no longer capture anything from the login command.

### Fixed

- Host icon resolution now folds `assetForHost(h).name` / `type_short` / `vendor` / `model` into the candidate + keyword-scan hay (#669). Pre-fix `hostIconUrl(h)` walked `label + id + host` only — operator clearing the display label (which falls back to asset.name per #621) silently dropped the primary brand keyword signal, leaving cleared-label rows with the wrong icon or none. Asset lookup is cheap (`assetForHost` reads from an indexed cache) and falls back to original behaviour when no asset matches. Operators relying on the asset record's name carrying brand info ("Synology RS820+", "Dell PowerEdge R740") now see the correct icon resolve automatically without re-typing the label.
- Ping port + transport per-host inputs now also disable when the host's main "enabled" is OFF (#668). Pre-fix only the Ping enable checkbox itself was gated on `!row.enabled` (#660); the dependent port + transport inputs were inside `x-show="!!(row.ping && row.ping.enabled)"` so unticking the host's main enable AFTER having ping enabled left the port + transport rows visible (because `row.ping.enabled` is still True in state until save) and editable. Both `x-show` extended to `!!(row.enabled && row.ping && row.ping.enabled)` and both `:disabled` extended to `isReadonly() || !row.enabled`. Same gating applied to the SNMP per-host inputs (target / community / version / port) for consistency.
- `loss_pct` format spec in `logic/ping_sampler.py` is now defensive — `(result.get('loss_pct') or 0):.0f` instead of `result.get('loss_pct'):.0f` (MED-008 / #667). Synthesized error path always sets 100.0 so the bug is latent today, but a future "sample taken, loss unknown" probe that returns None would have raised TypeError on the format spec. Forward-looking guard.
- WebAuthn `verify_authentication` 0/0 sign-counter check comment rewritten to match actual code behaviour (MED-002 / #662). Pre-fix the comment said "0/0 case is left to us" but the code skips when EITHER side is zero — actually delegated to the lib's contract (Duo's webauthn 2.x rejects 0→0 clones at response-verification). New comment documents the delegation + the per-credential `ever_nonzero` bookkeeping that would be needed if a future minor relaxes that contract. Lightweight fix; full schema change deferred until a real regression hits.
- `get_credential_by_credential_id` SELECT in `logic/auth.py` now includes `rp_id` (MED-003 / #663). Pre-fix the helper was schema-out-of-sync after #605's additive ALTER added the column. Login-finish path doesn't currently consume the field but a future caller wanting to detect "credential matches login but not current page" should read this rather than a separate query. Same drift class as `host_failure_state.last_failure_ts` regression #461 — every additive ALTER means audit every SELECT against that table.
- `extract_storage` unit-normalisation heuristic for hrStorageType=RAM (MED-004 / #664). Pre-fix `logic/snmp.py` treated `hrStorageAllocationUnits × hrStorageSize` as straight bytes. Older Cisco IOS-XE reports `Units=1, Size=<KB-count>` for RAM (Units 1, Size 2048000 = 2 GB → naive math stored 2 MB). Heuristic added: hrStorageType=RAM with `unit_bytes == 1 and total_bytes < 16 MiB` → treat Units as 1024 and re-multiply. Conservative cutoff because no real SNMP-speaking device on a modern fleet has < 16 MiB RAM, so the heuristic only kicks in when the math is obviously wrong.
- `_snmp_get` / `_snmp_walk` now log exception type at WARNING + carve out cancellation (MED-005 / #665). Pre-fix `logic/snmp.py` swallowed ALL exceptions and returned `{}`, so a misformatted OID or pysnmp internal bug surfaced as "host returned no data" — the partial-data fallthrough then armed the cooldown, leaving the operator chasing a "host not responding" symptom for a code-bug cause. Fix: `except (asyncio.CancelledError, KeyboardInterrupt): raise` BEFORE the broad except so lifespan cancellation propagates correctly; one `[snmp] WARN GET error...` / `[snmp] WARN WALK error...` line per error includes the exception type so debugging distinguishes "real network failure" from "code bug".
- Renamed `for c in creds:` → `for cred in creds:` in `api_local_login_webauthn_start` (MED-006 / #666). Outer `with db_conn() as c:` block already exits before the loop so it's technically correct, but the name reuse shadowed `main.py`'s convention (`c` = sqlite connection) and added reader-hazard. Trivial rename + comment documents the rationale so future contributors don't re-introduce the shadow.
- Lazy IO observer fan-out now honours `tuning_hosts_parallel_fetch` concurrency cap (MED-001 / #661). Pre-fix `_ensureHostRowObserver`'s entry-callback fired `refreshHostRow(id)` directly with no semaphore / throttle, bypassing the same cap that `loadHosts`' poll-driven path honoured. On a 200-host fleet, fast-scrolling could fan out 50+ concurrent `/api/hosts/one/{id}` requests in <500 ms, defeating the operator-set tunable. Fix: observer entries now collect into `_hostObserverPending` Set + 200 ms debounced flush; flush hands the queued ids to a new shared `_runHostRefreshQueue(ids)` worker that uses the SAME PARALLEL cap (per-call read of `me.client_config.hosts_parallel_fetch`, fallback 6). Both call sites now share one concurrency-discipline path.
- Host-level "enabled" checkbox now hard-gates every per-provider checkbox in Admin → Hosts editor (#660). Operator-reported: a host marked not-enabled (the row's master `enabled` toggle) could still have SSH / Ping / SNMP individually enabled, contradicting the row-level disable intent. Three checkboxes (`row.ssh.enabled`, `row.ping.enabled`, `row.snmp.enabled`) now compute `:checked` as `!!(row.enabled && row.<x>.enabled)` and `:disabled` as `isReadonly() || !row.enabled`; their containing labels add `:class="!row.enabled ? 'opacity-50' : ''"` so the visual signal matches the gated state. Save-time `saveHostsConfig` strips per-provider `enabled` keys when `h.enabled === false` so reload comes back consistent. Backend `_save_hosts_config` mirrors the strip as defence-in-depth — a malformed POST or any non-SPA caller can't persist `enabled: true` on a provider sub-dict for a disabled host. SSH dependent inputs (user / port / fqdn / password) extend their `:disabled` to also gate on `!row.enabled` so editing those fields is impossible while the host is off.
- SNMP `tuning_snmp_probe_timeout_seconds` + `tuning_snmp_concurrency` now actually consumed (#655). Pre-fix both knobs were declared in `logic/tuning.py:TUNABLES`, rendered in Admin → Config form rows with i18n labels, but NOT read at any of the three `_snmp.probe_snmp` call sites — the gather fan-out at `logic/gather.py` had no semaphore either. Operator moved sliders, hit Save, saw a green toast, value never reached SNMP. Fix: per-call reads via `tuning.tuning_int("tuning_snmp_probe_timeout_seconds")` at `_merge_one_host`'s SNMP block (`main.py`), at the gather fan-out (`logic/gather.py`), and at the test endpoint (`api_snmp_test`, with `max(10s, tunable)` so a tiny tunable doesn't cripple manual smoke probes). Concurrency cap lands as `asyncio.Semaphore(tuning_int("tuning_snmp_concurrency"))` wrapping the gather's `_one_snmp` calls. Audit recipe now in CLAUDE.md so the same drift class doesn't bite a future provider.
- SNMP tunables added to `SettingsIn` Pydantic model (#656). Pre-fix `tuning_snmp_probe_timeout_seconds` / `tuning_snmp_concurrency` were absent from `SettingsIn`, and Pydantic v2's default `extra="ignore"` silently dropped them on POST `/api/settings`. Save loop walks `tuning.TUNABLES.items()` calling `getattr(s, _k, None)` — for missing model attrs `getattr` returns None and the loop continues without persisting the value. Operator UX bug (visible green-toast lie pattern from #455's BUG-001 — same drift class, different layer). Fix: two `Optional[str]` declarations next to the ping ones at `main.py:1580`, plus the new SNMP-cache-TTL pair from #659.
- SNMP per-host probe targeting now hard-gates on alias OR `snmp_name` — no bare-id fallthrough (#657). Pre-fix `_merge_one_host`'s SNMP block (`main.py:3956-3960`) and `logic/gather.py:917-921` resolved target as `snmp_aliases[h.id] OR row.snmp_name OR h.id` — the bare `h.id` final fallback meant a host with NEITHER an alias NOR `snmp_name` set still got probed against its hostname-as-target. Net effect on a 200-host fleet with SNMP fleet-enabled: 200 SNMP probes per tick, ~all-but-mapped of which timed out (~5s × 200 = 1000s on first cold tick before the cool-down kicked in). Fix: `if not (alias or snmp_name): skip` — no bare-hostname fallback. Coupled with #655's concurrency cap, the first SNMP-enable tick now fires probes ONLY against the operator's curated SNMP rows.
- SNMP `probe_snmp` `try/except asyncio.TimeoutError` now reachable (#658). Pre-fix `logic/snmp.py:771-781` wrapped a 13-fan-out `asyncio.gather` in a `try/except asyncio.TimeoutError` block, but `asyncio.gather` cannot raise `TimeoutError` without `asyncio.wait_for` — dead code. Cool-down arming through that branch never fired. Fix: wrap the gather in `asyncio.wait_for(timeout=(timeout + 2) * 2)` so the catch is reachable AND the caller earns a wall-clock guarantee. Also added `except (asyncio.CancelledError, KeyboardInterrupt): raise` BEFORE the broad `except Exception` so lifespan shutdown's task cancellation propagates correctly. New CLAUDE.md rule documents that `try/except asyncio.TimeoutError` around a bare gather is dead code; future authors get the diagnosis up front.
- SNMP `/api/snmp/test` endpoint now whitelist-validates the version field (MED-007). Pre-fix any string flowed through to `_build_auth_data`'s "unknown version → fall back to v2c" path, so an operator typing `v1` saw "v2c worked" — confusing, hides config errors. Now returns 400 on anything other than `v2c` or `v3`, matching the settings save-path validator.

Net effect on a 200-host fleet with 20 visible: probe count drops from ~150 to ~20 per page load, with proportional savings on every poll cycle.

Highlights: NE-only host charts nowpopulate Disk I/O on Linux (`node_disk_*`) AND FreeBSD (`node_devstat_*`),distinguish "host is idle" from "exporter doesn't expose this collector"(#327), pull cross-provider snapshots so existing data survives a provideroutage (#222 series), and fold a refreshed pile of UX polish (host-drawer
freshness label, scroll-on-expand, action-bar sticky behaviour, login-page
logo, SSH terminal cols/rows). Backend / docs / refactor work was largely
internal cleanup — see the Internal section.

### Internal

- Documentation moved from `notes/guidelines/` and `notes/RELEASE_PROCESS.md` to a new `docs/` directory; new `docs/screenshots/` for README images; new `docs/README.md` index. Operator-private files (`note_todo.txt`, `notes.txt`, `forgejo_runner_config.yml`, the live Grafana dashboard) stay in `notes/`. `CHANGELOG.md` and the root `README.md` keep their root-level positions per convention. Code docstrings / cross-references updated to the new paths.

- Consolidated `_load_curated_hosts` between the two NE samplers — both now import the canonical `curated_ne_hosts()` from `logic/db.py` (#332 /). Drops ~30 duplicated lines and means a future NE-aware sampler (e.g. ping / SNMP) only adds to the canonical helper.
- New `_format_provider_test_summary()` in `main.py` keeps the Pulse + Beszel test-connection response shape identical (#334 /). Webmin and Portainer keep their bespoke summaries; future `{hosts: {...}}`-shaped providers should reuse the helper.

- Replaced operator-private hostnames in shipped docs and code comments with example.com placeholders (#337).

### Changed

- `Settings → Portainer → Test` now validates the configured endpoint id, not just `/api/status` (#335 / DEAD-002). Test now probes `/api/endpoints/{endpoint_id}` after the status check; success message reads `OK — Portainer X.Y.Z, endpoint <Name> reachable`, and a misconfigured endpoint id surfaces as `endpoint X not found on this Portainer` instead of failing silently until the next gather. Falls back to the saved `endpoint_id` when the form's value is blank.

### Added

- Host-drawer charts now show a subtle `Updated Xs/m/h ago` freshness hint beside the time-range picker (#338). Stamped on every successful chart fetch and ticks every second so the seconds digit counts visibly. Hidden until the first response lands.

### Changed

- Show Debug + SSH-run toggles in the host drawer now scroll the just-expanded body to the top of the drawer viewport (#339). Walks up to the drawer's scrollable ancestor explicitly and sets `scrollTop` so Safari (which sometimes scrolled the page instead of the drawer with bare `scrollIntoView`) tracks correctly.

### Fixed

- Admin → Hosts / Host groups action bar now matches the editor section's width AND pins correctly to the viewport bottom while scrolling (#340). Three-iteration fix: introduced `--app-footer-clearance: 48px` token to clear the sticky footer, then switched `.hosts-config-actionbar` from `position: fixed` (which overshot the section because the admin layout nests it inside a 1100px-max-width `.page-layout` with a 220px sidebar) to `position: sticky; bottom: var(--app-footer-clearance)` so the bar inherits the section's natural in-flow width. Sticky was silently broken by `html, body { overflow-x: hidden }` — that property promotes html/body to a scroll container, which sticky descendants pin to instead of the viewport. Swapped to `overflow-x: clip` (clips overflow without establishing a scroll container) so sticky operates against the viewport as expected.

- SSH terminal modal: xterm cols/rows now match the modal's actual dimensions on first open even when xterm's `FitAddon.proposeDimensions()` silently returns `undefined`. New `measureAndResize` helper tries FitAddon first; if `term.cols` stays at the default 80, falls back to a manual `getBoundingClientRect()` measurement using known cell metrics (~7.85px × ~17.5px per cell at 13px Menlo / Consolas / DejaVu Mono) and calls `term.resize()` directly. Helper runs on rAF + 50/250/600/1200ms `setTimeout`s + a `ResizeObserver` + the WS `ready` control frame.

- Login-page logo no longer shows a white halo at the rounded corners (#336). `static/login.html` swapped from the rasterised `icon-512.png` to `omnigrid.svg`, and `.login-logo` lost its redundant `background: var(--surface)` fill. The SVG renders with crisp anti-aliasing at any zoom level; favicon keeps the PNG for universal browser compat.

- SSH terminal modal: xterm cols/rows now match the modal's actual dimensions on first open. The initial `fit()` was running before the flex-1 `.terminal-host` had its layout committed, so xterm fell back to the default 80×24 and the shell wrapped mid-line. Fit now fires through a belt-and-braces staircase — double `requestAnimationFrame` + `setTimeout` retries at 50 / 250 / 600 ms + a `ResizeObserver` on the host element + a final refit when the WS `ready` control frame lands. `fit.fit()` is idempotent so the overlap is a no-op when an earlier pass already produced the right size.

### Changed

- Host drawer "No NIC activity" hint now branches on whether node-exporter is in play, not on whether Beszel is mapped (#321). Hybrids that run BOTH a Beszel agent (reporting zero NICs) AND an NE exporter (with real net data) — opnsense was the canonical example — now see the NE-flavoured wording (loopback / Docker bridges / veth pairs excluded from totals) instead of the Beszel "set NICS=eth0" hint, which was misleading because NE was the source the operator expected to fix. Pure Beszel hosts still see the NICS hint.

- NE-only host drawer: Disk I/O + Network charts now distinguish "no activity in window" from "node-exporter doesn't expose this collector" (#327). Backend `/api/hosts/history` (NE path) now returns a `collectors` dict — `{disk_io, net, fs, mem, cpu}` booleans recording whether any sample in the window held a non-null value for each metric. Frontend gates a new "Enable the diskstats / netstats collector" empty-state ahead of the existing idle copy so hosts whose exporter is permanently missing the collector get remediation guidance instead of a wait-and-see message. New `hosts_extra.collectors_missing.*` i18n family.

### Added

- FreeBSD Disk I/O support for NE-only hosts (#331). `parse_disk_counters` now falls back to `node_devstat_bytes_total{device,type}` when the Linux `node_disk_*` family produces no eligible devices, so opnsense / pfSense / TrueNAS / FreeBSD hosts populate the Disk I/O chart from the same sampler pipeline. FreeBSD-specific exclusions: `pass*` (SCSI passthrough), `md*` (memdisk), `cd*`. Linux pass takes precedence when both families are present; smoke test covers parse + rate + precedence.

- Admin → Host groups tab: pagination + sticky action bar mirroring the Hosts editor (#328). Page size persists to localStorage; Add / Collapse all / Save / scroll-to-Top stay pinned to the viewport on long lists. Action bar repositioned `position: fixed; bottom` so it's visible from page entry instead of only after scrolling past its natural position; new `.hosts-config-page-bottom-pad` class gives both editors a 80px bottom gutter so the fixed bar can never obscure the last row.

New `parse_disk_counters` in `logic/node_exporter.py`, `host_metrics_samples` table gained `disk_read_bps` / `disk_write_bps` columns, sampler tracks rates independently from net (a disk subsystem reboot doesn't drop net rates and vice-versa). Hosts whose exporter has the diskstats collector enabled show real I/O after ~10 minutes (two sampler ticks).
