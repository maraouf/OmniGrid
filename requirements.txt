# Optional ICMP transport for the Ping host-stats provider.
# logic/ping.py imports this LAZILY and falls back to TCP-connect
# probes when the package is missing — so this dep is genuinely
# optional from a correctness standpoint. We pin it anyway so the
# Settings → Host stats → Ping panel's "use ICMP" toggle is wired

# on the container (wired via ``cap_add: [NET_RAW]`` on the
# `omnigrid` service in docker-compose.yml). Without the cap the
# import succeeds but every probe raises SocketPermissionError →
# ping.py auto-falls-back to TCP at probe time, so a missing cap is
# not a boot-blocker, just "ICMP toggle on but TCP is what runs".
